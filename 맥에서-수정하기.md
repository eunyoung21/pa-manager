# 노트북(맥)에서 PA 매니저 수정하기

회사 PC와 노트북 둘 다에서 코드를 고칠 수 있다. 코드는 GitHub에 있고, 데이터(실제)는
Render의 Supabase에 있다. 노트북에선 시크릿 없이 로컬 사본으로 안전하게 작업한다.

## ① 처음 한 번만 (맥 셋업)
```
# 1. Node.js 설치: https://nodejs.org (LTS) — 한 번만
# 2. 코드 받기
git clone https://github.com/eunyoung21/pa-manager.git
cd pa-manager
npm install
```

## ② 수정할 때마다 (반복)
```
# 1) 최신 코드 먼저 받기 (회사PC에서 바꾼 게 있을 수 있으니 꼭!)
git pull

# 2) 서버 켜고 테스트
npm start
#   → 브라우저 http://localhost:3456, 로그인 admin / admin1234
#   → 코드 고치면 Ctrl+C 로 끄고 npm start 다시 (자동 새로고침 없음)

# 3) 다 됐으면 저장 → 실제 사이트 반영
git add -A
git commit -m "무엇을 고쳤는지"
git push
#   → Render가 자동 배포 (1~2분 뒤 실제 사이트에 반영)
```

## ⚠️ 기억할 것
- **시작 전 `git pull` 필수.** 회사PC·노트북 둘 다 쓰니 안 받고 고치면 충돌. 끝나면 바로 `push`.
- **로컬은 연습용 사본.** 노트북에서 보는 `data.json`은 실제 Render 데이터가 아니라 깃에 든 복사본.
  여기서 데이터를 바꿔도 실제 사이트엔 영향 없음. (코드만 push하면 반영됨)
- **로컬 로그인은 `admin / admin1234`** — 실제 사이트 비번과 다름. 정상.
- 고친 게 맘에 안 들면 push 전엔 `git checkout .` 로 되돌리기.
